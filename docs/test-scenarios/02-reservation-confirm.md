## 단계별 체크

### 1단계 — 예약 정보 화면 (Screen 2)

- [ ] 상단 헤더 "방문 예약 안내"
- [ ] 빨간 박스 안내문 ("예정된 방문 작업을 확인해 주세요")
- [ ] 흰색 카드 + 상단 "방문 예약 정보" 제목 + 구분선
- [ ] InfoRow 3개:
  - [ ] 방문 예정일 (`YYYY년 M월 D일 (요일) HH:00 ~ HH:00`) — stack layout (label 위)
  - [ ] 방문 시간대 (`HH:00 ~ HH:00`) — row layout
  - [ ] 작업 종류 — 오더별 멀티라인 (`{타입라벨} - {상품명}` 한 줄씩)
- [ ] "⚠ 방문 1일 전까지 변경 가능합니다" 안내
- [ ] CSNote 보임
- [ ] 하단 고정 영역: `[예약 변경]`(빨간 테두리 흰 배경) / `[예약 확정]`(빨강 메인) 두 버튼

### 2단계 — ConfirmDialog

- [ ] `[예약 확정]` 클릭 → 중앙 다이얼로그 표시 (fade-in 애니메이션)
- [ ] 다이얼로그 제목 "예약을 확정하시겠습니까?"
- [ ] 설명에 날짜·시간 두 줄로 표시
- [ ] `[취소]` / `[확정]` 두 버튼
- [ ] 백드롭(검은 반투명) 클릭 시 다이얼로그 닫힘
- [ ] ESC 키 누르면 닫힘
- [ ] 다이얼로그 떠 있을 때 본문 스크롤 잠금

### 3단계 — 확정 처리

- [ ] `[확정]` 클릭 → 다이얼로그 버튼에 스피너 + 양쪽 버튼 disabled
- [ ] mock 응답 받으면 다이얼로그 닫히며 완료 화면으로 stage 전환 (URL 그대로)

### 4단계 — 완료 화면 (Screen 3)

- [ ] 상단 헤더 "예약 확정"
- [ ] CheckIcon (성공 컬러)
- [ ] "예약이 확정되었습니다" 큰 글씨
- [ ] "방문 당일 예약 정보를 별도로 안내드립니다" 안내
- [ ] 회색 카드 안에 정보 표시:
  - [ ] 방문 예정일 + 시간 (row)
  - [ ] 방문 시간대 (row)
  - [ ] 작업 종류 (멀티라인)
- [ ] 하단 [확인] 버튼 **없음** (의도된 제거 — 카톡 자체 닫기 유도)

## 분기 시나리오

### [예약 변경] 버튼 클릭

- [ ] `[예약 변경]` 누르면 `/login`으로 이동
- [ ] **현재 동작 확인**: state.from 안 넘김 → 진입 가드에서 차단되어 /error 갈 가능성
  - 의도된 동작이면 OK / 아니면 from 전달 필요

## 회귀 체크

- [ ] 완료 화면 후 브라우저 ← 백버튼 → 카톡(이전 페이지)으로 빠짐 (확정 화면으로 안 돌아감)
- [ ] 새로고침 시 진입 화면(Screen 2)부터 다시 시작 (stage state 휘발)
- [ ] 멀티 오더 케이스: 작업 종류가 줄바꿈으로 여러 줄 깔끔하게
- [ ] 데스크탑/모바일 가로폭 정상
