# User Flow

카카오 알림톡 웹뷰 사용자 시나리오. 알림톡 종류별로 묶음.


## 사전방문안내 알림톡

방문 예정 고객에게 사전에 발송되는 알림톡. 본문 내 두 개의 버튼(예약 확정 / 예약 변경)으로 분기.

### 시나리오 1. 예약 확정 (OTP 불필요)

```
[사용자가] 사전 방문 안내 알림톡을 받음
  → 예약 확정 버튼 클릭
  → 예약 확정 화면이 바로 뜸
  → 확정 버튼 누름
  → 확정 완료 안내
```

### 시나리오 2. 예약 변경 (OTP 필요)

```
[사용자가] 사전 방문 안내 알림톡을 받음
  → 예약 변경 버튼 클릭
  → 본인인증 화면이 뜸 (마스킹된 본인 번호 보임)
  → OTP 입력
  → 변경 가능한 날짜/시간 목록이 뜸
  → 새 일정 선택
  → 변경 완료 안내
```


## 당일방문안내 알림톡

방문 당일 발송되는 알림톡. 작업자 정보까지 함께 표시.

### 시나리오 3. 예약 상세 조회 (OTP 필요)

```
[사용자가] 당일 방문 안내 알림톡을 받음
  → 예약 상세 조회 버튼 클릭
  → 본인인증 화면이 뜸 (마스킹된 본인 번호 보임)
  → OTP 입력
  → 예약 상세 정보가 뜸
     - 오더 상태 (진행상태)
     - 예약 일시
     - 작업자 정보 (이름, 연락처, 사진)
```


## 조치방법 안내

인터넷 사용 중 오류 발생 시 고객이 스스로 따라할 수 있는 셀프 트러블슈팅 가이드. 오더 무관.

### 시나리오 4. 조치방법 (OTP 불필요)

```
[사용자가] 조치방법 안내 링크에 진입
  → 단계별 가이드가 뜸 (모뎀 재부팅, 케이블 확인 등)
  → 각 단계 따라하며 완료 체크
  → 해결됨 → 종료
  → 해결 안 됨 → 상담/방문 신청 안내
```
